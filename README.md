# AMD Prime Discovery Dashboard (Visualisation & Bridge Suite)

A real-time single-page web dashboard and Socket.IO bridge server that visualizes high-performance mathematical results generated by the C++ GPU Prime Gap Engine.

---

## 🎨 Features & Visuals

* **Real-time Stats**: Prime discovery counts, speed (primes/sec), and maximum prime gap tracked and displayed live.
* **Prime Gap Scatter Plot**: Real-time rendering of prime gaps overlaid with Cramér's Conjecture logarithmic prediction.
* **Interactive Ulam Spiral**: Live-generated 2D coordinate plot tracking the location of primes, with full mouse zoom, pan, and hover support.
* **Gap Frequency Histogram**: Compares gap distribution frequency with a mathematical Gumbel extreme value distribution curve.
* **Chebyshev Bias Trend**: Charts the cumulative race between primes $p \equiv 1 \pmod 4$ and $p \equiv 3 \pmod 4$.
* **Discovery Feed & Audio Cues**: Chronological log displaying Cunningham chains, prime constellations, and new record-breaking gaps with audio notifications.
* **Copy & Share**: Quick button to generate a clean markdown summary of current findings to the clipboard.

---

## ⚙️ Backend Bridge Architecture (`server.js`)

The Node.js backend performs three critical tasks:
1. **File Tailing**: Watches `/tmp/prime_results.json` (or OS temp directory path), reads newly appended bytes, parses the lines, and normalizes key names.
2. **Socket.IO Broadcasting**: Emits normalized events (`new_data`) to all connected browser clients.
3. **Mock Simulation (Fallback)**: If the real C++ results file does not exist yet, the server automatically starts a background simulation loop generating math-valid data. Once the C++ engine writes to the file, the simulator shuts down and real-time streaming takes over.

## 🚀 Running the Dashboard

### Prerequisites
* **Node.js** (v18.x or newer recommended)
* **npm**

### Installation
Install the project dependencies (Express, CORS, Socket.IO, Concurrently) by running:
```bash
npm install
```

### Running Locally
To launch both the Node.js Socket.IO server and host the static dashboard frontend concurrently:
```bash
npm run dev
```
* **Frontend Dashboard**: Opened at [http://localhost:3000](http://localhost:3000) (via http-server)
* **Socket.IO Bridge Server**: Listening on [http://localhost:5000](http://localhost:5000)

Alternative scripts (available in `package.json`):
* `npm start` / `npm run backend`: Starts just the Node.js Socket.IO bridge server.
* `npm run frontend`: Serves the static HTML/CSS/JS frontend page on port 3000.

## 🔌 API Endpoints

### 1. Health Status (`GET /health`)
Check connection status and active client count:
```bash
curl http://localhost:5000/health
```

### 2. Manual Mock Data Inject (`POST /inject`)
For testing specific events without the compute engine, you can post payloads directly:
```bash
curl -X POST http://localhost:5000/inject \
  -H "Content-Type: application/json" \
  -d '{"type": "gap", "gap": 72, "left": 1000000000061, "right": 1000000000133}'
```
Valid payload types are: `prime`, `gap`, `chain`, `tuple`, `ap`, `chebyshev`, `ulam_batch`, and `stats`.
